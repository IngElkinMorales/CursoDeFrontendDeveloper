Unidades de Medida en CSS

  • Píxeles (px): Es la unidad de medida más común en CSS. Un píxel es la unidad de medida más pequeña en una pantalla y es una medida fija y específica de tamaño.

  • Porcentaje (%): Es una unidad de medida relativa que se basa en el tamaño del elemento padre. Por ejemplo, si un elemento hijo tiene un ancho del 50% en relación con el ancho del elemento padre, entonces su ancho será la mitad del ancho del elemento padre.

  • EM: Es otra unidad de medida relativa que se basa en el tamaño de la fuente del elemento padre. Por ejemplo, si un elemento tiene una altura de 2em y el tamaño de fuente de su elemento padre es de 16 píxeles, entonces la altura del elemento será de 32 píxeles.

  • REM: Es una unidad de medida relativa similar a EM, pero se basa en el tamaño de fuente del elemento raíz (es decir, el elemento HTML). Esto hace que REM sea más predecible y fácil de usar que EM.

  • VW y VH: Estas son unidades de medida relativas que se basan en el tamaño de la ventana del navegador. VW (Viewport Width) se refiere al ancho de la ventana del navegador y VH (Viewport Height) se refiere a la altura de la ventana del navegador. Por ejemplo, si un elemento tiene un ancho del 50vw, entonces su ancho será la mitad del ancho de la ventana del navegador.

  • PT: Es una unidad de medida absoluta que se utiliza principalmente para la impresión. Un punto es equivalente a 1/72 de una pulgada.

    • ¿Qué son y para qué nos sirven las arquitecturas CSS?

    Las arquitecturas en CSS son conjuntos de convenciones y reglas que se utilizan para organizar y estructurar el código CSS de un proyecto. Estas arquitecturas tienen como objetivo mejorar la mantenibilidad, escalabilidad, y facilidad de trabajo en equipo del código CSS. Algunas de las arquitecturas más comunes en CSS son:

  • BEM (Block Element Modifier): Esta arquitectura se basa en la idea de dividir la interfaz en bloques (blocks), elementos (elements) y modificadores (modifiers). Cada bloque es un componente independiente, compuesto por elementos y modificadores. La nomenclatura de BEM se basa en un sistema de nomenclatura de clases que describe la relación entre los diferentes bloques, elementos y modificadores.
  • SMACSS (Scalable and Modular Architecture for CSS): SMACSS es una arquitectura que se centra en la modularidad, la escalabilidad y la reutilización de componentes. Se basa en la idea de dividir la hoja de estilos en cinco categorías: base, diseño, módulos, estados y temas.
  • OOCSS (Object Oriented CSS): OOCSS es una arquitectura que se basa en la idea de que los estilos de un sitio web se deben tratar como objetos con propiedades y comportamientos. Se enfoca en la separación de la estructura y el diseño, la reutilización y la modularidad.
  • Atomic CSS: Esta arquitectura se basa en la creación de clases de CSS muy específicas y pequeñas, que representan características visuales atómicas (por ejemplo, tamaño de fuente, color de fondo, etc.). Luego, estas clases se pueden combinar y reutilizar para construir estilos más complejos.

    • Cada arquitectura tiene sus propias convenciones y reglas para organizar y estructurar el código CSS. El objetivo de todas estas arquitecturas es mejorar la organización y la mantenibilidad del código, lo que puede ahorrar tiempo y esfuerzo en proyectos grandes y complejos, y también facilitar el trabajo en equipo y la colaboración en proyectos de desarrollo web.
    Algunas buenas prácticas son:

  • Lineamientos: Hace referencia a los dirección que establece un equipo de desarrollo para construir el código.
  • Documentación: Es el registro que debe existir para que personas del proyecto o personas nuevas que se unen puedan entender de manera clara la forma de trabajo y la estructura del código de manera más sencilla.
  • Estándares: Son las normas, patrones y similares que se establecen para construir el código.
